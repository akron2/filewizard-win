# Task: Fix Transcription Model Download & Selection UI

## Контекст (Context)
Проект - FastAPI приложение с фоновыми задачами. Пользователь загружает медиафайлы и выбирает модель транскрипции (Whisper) или диалогизации (pyannote).

## Проблема (Issue)
Выбор модели для транскрипции/аудио не работает корректно в тех случаях, когда требуемая модель не скачана локально или требует авторизации. 
Вместо того чтобы скачать модель (и показать прогресс загрузки) или запросить авторизацию (например, токен HuggingFace), процесс завершается с ошибкой или зависает. В UI (фронтенде) нет индикации скачивания, а окно авторизации не всплывает.

## Задача программиста (Your Task)
1. Найдите логику инициализации моделей транскрипции (`faster_whisper.WhisperModel`) и диалогизации в `main.py` (или связанных модулях).
2. Реализуйте проверку наличия выбранной модели до начала транскрипции (насколько это возможно).
3. При скачивании модели отправляйте через WebSocket статус, например `{ "status": "downloading_model", "message": "Downloading whisper model..." }`, чтобы фронтенд мог отображать спиннер или прогресс бар.
4. Если скачивание падает с HTTP 401/403 (например, требуется токен HuggingFace), фоновый джоб должен бросить специфичную ошибку, которая через WebSocket дойдет до фронтенда. Фронтенд должен показать модальное окно с запросом токена. Требуется реализовать UI-компонент, который умеет запрашивать токен.
5. Обновите вызов транскрипции так, чтобы он корректно принимал сохраненный на клиенте/сервере токен авторизации и мог осуществлять скачивание. 

Твоя цель: обеспечить прозрачный для пользователя процесс загрузки моделей и gracefully (мягко) обрабатывать ситуации с отсутствием прав доступа (токена), запрашивая их в UI.
